## <a name="basic-grouping"></a><span data-ttu-id="8ae12-111">基本分組</span><span class="sxs-lookup"><span data-stu-id="8ae12-111">Basic grouping</span></span>

<span data-ttu-id="8ae12-112">在 Azure 中，[資源群組](/azure/azure-resource-manager/resource-group-overview#resource-groups)是以邏輯方式將訂用帳戶內的資源分組的核心資源組織機制。</span><span class="sxs-lookup"><span data-stu-id="8ae12-112">In Azure, [resource groups](/azure/azure-resource-manager/resource-group-overview#resource-groups) are a core resource organization mechanism to logically group resources within a subscription.</span></span>

<span data-ttu-id="8ae12-113">資源群組會做為具有一般生命週期或共用管理條件約束 (例如原則或以角色為基礎的存取控制 (RBAC) 要求) 的資源容器。</span><span class="sxs-lookup"><span data-stu-id="8ae12-113">Resource groups act as containers for resources with a common lifecycle or shared management constraints such as policy or role-based access control (RBAC) requirements.</span></span> <span data-ttu-id="8ae12-114">資源群組不能建立巢狀結構，且資源只能屬於一個資源群組。</span><span class="sxs-lookup"><span data-stu-id="8ae12-114">Resource groups can't be nested, and resources can only belong to one resource group.</span></span> <span data-ttu-id="8ae12-115">某些動作可套用於資源群組中的所有資源。</span><span class="sxs-lookup"><span data-stu-id="8ae12-115">Some actions can act on all resources in a resource group.</span></span> <span data-ttu-id="8ae12-116">例如，刪除資源群組即可移除該群組內的所有資源。</span><span class="sxs-lookup"><span data-stu-id="8ae12-116">For example, deleting a resource group removes all resources within that group.</span></span> <span data-ttu-id="8ae12-117">有幾種常見模式可建立資源群組，通常區分為兩種類別：</span><span class="sxs-lookup"><span data-stu-id="8ae12-117">There are common patterns when creating resource groups, commonly divided into two categories:</span></span>

- <span data-ttu-id="8ae12-118">傳統 IT 工作負載：最常見的分組方式為依照相同生命週期內的項目分組，例如應用程式。</span><span class="sxs-lookup"><span data-stu-id="8ae12-118">Traditional IT workloads: Most often grouped by items within the same lifecycle, such as an application.</span></span> <span data-ttu-id="8ae12-119">依照應用程式分組，即可進行個別應用程式管理。</span><span class="sxs-lookup"><span data-stu-id="8ae12-119">Grouping by application allows for individual application management.</span></span>
- <span data-ttu-id="8ae12-120">敏捷式 IT 工作負載：著重於外部客戶面向的雲端應用程式。</span><span class="sxs-lookup"><span data-stu-id="8ae12-120">Agile IT workloads: Focus on external customer-facing cloud applications.</span></span> <span data-ttu-id="8ae12-121">資源群組通常會反映出部署 (例如 Web 層或應用程式層) 和管理的功能層次。</span><span class="sxs-lookup"><span data-stu-id="8ae12-121">These resource groups often reflect the functional layers of deployment (such as web tier or app tier) and management.</span></span>

## <a name="deployment-consistency"></a><span data-ttu-id="8ae12-122">部署一致性</span><span class="sxs-lookup"><span data-stu-id="8ae12-122">Deployment consistency</span></span>

<span data-ttu-id="8ae12-123">Azure 平台建置在基底的資源群組機制之上，為使用範本來將您的資源部署到雲端環境提供系統。</span><span class="sxs-lookup"><span data-stu-id="8ae12-123">Building on top of the base resource grouping mechanism, the Azure platform provides a system for using templates to deploy your resources to the cloud environment.</span></span> <span data-ttu-id="8ae12-124">您可以在部署工作負載時使用範本建立一致的組織和命名慣例，強制執行您資源部署和管理設計的各個層面。</span><span class="sxs-lookup"><span data-stu-id="8ae12-124">You can use templates to create consistent organization and naming conventions when deploying workloads, enforcing those aspects of your resource deployment and management design.</span></span>

<span data-ttu-id="8ae12-125">[Azure Resource Manager 範本](/azure/azure-resource-manager/resource-group-overview#template-deployment)可讓您使用預先決定的組態和資源群組結構，以一致的狀態重複部署資源。</span><span class="sxs-lookup"><span data-stu-id="8ae12-125">[Azure Resource Manager templates](/azure/azure-resource-manager/resource-group-overview#template-deployment) allow you to repeatedly deploy your resources in a consistent state using a predetermined configuration and resource group structure.</span></span> <span data-ttu-id="8ae12-126">Resource Manager 範本可協助您定義一組標準作為部署基礎。</span><span class="sxs-lookup"><span data-stu-id="8ae12-126">Resource Manager templates help you define a set of standards as a basis for your deployments.</span></span>

## <a name="policy-consistency"></a><span data-ttu-id="8ae12-130">原則一致性</span><span class="sxs-lookup"><span data-stu-id="8ae12-130">Policy consistency</span></span>

<span data-ttu-id="8ae12-131">為確保在建立資源時套用治理原則，資源群組設計的一部份會涉及使用一般組態部署資源。</span><span class="sxs-lookup"><span data-stu-id="8ae12-131">To ensure that governance policies are applied when resources are created, part of resource grouping design involves using a common configuration when deploying resources.</span></span>

<span data-ttu-id="8ae12-133">例如，您可能需要讓訂用帳戶內部署的所有虛擬機器皆連線到由您的中央 IT 小組所管理的一般子網路。</span><span class="sxs-lookup"><span data-stu-id="8ae12-133">For example, you may have a requirement that all virtual machines deployed within your subscription connect to a common subnet managed by your central IT team.</span></span> <span data-ttu-id="8ae12-134">您可以建立標準範本，以用於部署會針對工作負載建立個別資源群組，並在其中部署必要 VM 的工作負載 VM。</span><span class="sxs-lookup"><span data-stu-id="8ae12-134">You can create a standard template for deploying workload VMs which would create a separate resource group for the workload and deploy the required VMs there.</span></span> <span data-ttu-id="8ae12-135">此資源群組會有一項原則規則，限制只允許將資源群組內的網路介面加入至共用子網路。</span><span class="sxs-lookup"><span data-stu-id="8ae12-135">This resource group would have a policy rule to only allow network interfaces within the resource group to be joined to the shared subnet.</span></span>

<span data-ttu-id="8ae12-136">如需有關在雲端部署內強制執行原則決策的更深入討論，請參閱[原則強制執行](../policy-enforcement/overview.md)。</span><span class="sxs-lookup"><span data-stu-id="8ae12-136">For a more in-depth discussion of enforcing your policy decisions within a cloud deployment, see [Policy enforcement](../policy-enforcement/overview.md).</span></span>

## <a name="hierarchical-consistency"></a><span data-ttu-id="8ae12-137">階層式一致性</span><span class="sxs-lookup"><span data-stu-id="8ae12-137">Hierarchical consistency</span></span>

## <a name="automated-consistency"></a><span data-ttu-id="8ae12-143">自動化的一致性</span><span class="sxs-lookup"><span data-stu-id="8ae12-143">Automated consistency</span></span>
