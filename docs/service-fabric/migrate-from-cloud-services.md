## <a name="why-microservices"></a><span data-ttu-id="bc15f-133">使用微服務的理由？</span><span class="sxs-lookup"><span data-stu-id="bc15f-133">Why Microservices?</span></span>

<span data-ttu-id="bc15f-134">本文不會深入討論微服務，但 Tailspin 希望藉由移往微服務架構而能獲得的優勢如下：</span><span class="sxs-lookup"><span data-stu-id="bc15f-134">An in-depth discussion of microservices is beyond scope of this article, but here are some of the benefits that Tailspin hopes to get by moving to a microservices architecture:</span></span>

- <span data-ttu-id="bc15f-135">**應用程式升級**。</span><span class="sxs-lookup"><span data-stu-id="bc15f-135">**Application upgrades**.</span></span> <span data-ttu-id="bc15f-136">服務可以獨立地部署，以便能夠採取累加方式升級應用程式。</span><span class="sxs-lookup"><span data-stu-id="bc15f-136">Services can be deployed independently, so you can take an incremental approach to upgrading an application.</span></span>
- <span data-ttu-id="bc15f-137">**復原功能和錯誤隔離**。</span><span class="sxs-lookup"><span data-stu-id="bc15f-137">**Resiliency and fault isolation**.</span></span> <span data-ttu-id="bc15f-138">如果服務失敗，其他服務會繼續執行。</span><span class="sxs-lookup"><span data-stu-id="bc15f-138">If a service fails, other services continue to run.</span></span>
- <span data-ttu-id="bc15f-139">**延展性**。</span><span class="sxs-lookup"><span data-stu-id="bc15f-139">**Scalability**.</span></span> <span data-ttu-id="bc15f-140">服務可以獨立調整。</span><span class="sxs-lookup"><span data-stu-id="bc15f-140">Services can be scaled independently.</span></span>
- <span data-ttu-id="bc15f-141">**彈性**。</span><span class="sxs-lookup"><span data-stu-id="bc15f-141">**Flexibility**.</span></span> <span data-ttu-id="bc15f-142">服務是圍繞商務案例 (而不是技術堆疊) 來設計的，因此可讓您更輕鬆地將服務移轉至新的技術、架構或資料存放區。</span><span class="sxs-lookup"><span data-stu-id="bc15f-142">Services are designed around business scenarios, not technology stacks, making it easier to migrate services to new technologies, frameworks, or data stores.</span></span>
- <span data-ttu-id="bc15f-143">**敏捷式開發**。</span><span class="sxs-lookup"><span data-stu-id="bc15f-143">**Agile development**.</span></span> <span data-ttu-id="bc15f-144">相較於單體式應用程式，個別服務的程式碼較少，因此其程式碼基底較容易理解、知道原由和進行測試。</span><span class="sxs-lookup"><span data-stu-id="bc15f-144">Individual services have less code than a monolithic application, making the code base easier to understand, reason about, and test.</span></span>
- <span data-ttu-id="bc15f-145">**小型焦點小組**。</span><span class="sxs-lookup"><span data-stu-id="bc15f-145">**Small, focused teams**.</span></span> <span data-ttu-id="bc15f-146">應用程式分成了許多小型服務，因此可由小型的聚焦小組建置各個服務。</span><span class="sxs-lookup"><span data-stu-id="bc15f-146">Because the application is broken down into many small services, each service can be built by a small focused team.</span></span>

## <a name="why-service-fabric"></a><span data-ttu-id="bc15f-147">使用 Service Fabric 的理由？</span><span class="sxs-lookup"><span data-stu-id="bc15f-147">Why Service Fabric?</span></span>
      
<span data-ttu-id="bc15f-148">Service Fabric 適合用於微服務架構，因為分散式系統所需的大部分功能都會內建在 Service Fabric 中，包括：</span><span class="sxs-lookup"><span data-stu-id="bc15f-148">Service Fabric is a good fit for a microservices architecture, because most of the features needed in a distributed system are built into Service Fabric, including:</span></span>

- <span data-ttu-id="bc15f-149">**叢集管理**。</span><span class="sxs-lookup"><span data-stu-id="bc15f-149">**Cluster management**.</span></span> <span data-ttu-id="bc15f-150">Service Fabric 會自動處理節點的容錯移轉、健康情況監視和其他叢集管理功能。</span><span class="sxs-lookup"><span data-stu-id="bc15f-150">Service Fabric automatically handles node failover, health monitoring, and other cluster management functions.</span></span>
- <span data-ttu-id="bc15f-151">**水平調整**。</span><span class="sxs-lookup"><span data-stu-id="bc15f-151">**Horizontal scaling**.</span></span> <span data-ttu-id="bc15f-152">當您將節點新增至 Service Fabric 叢集時，應用程式會自動調整，因為服務會分散到這些新節點。</span><span class="sxs-lookup"><span data-stu-id="bc15f-152">When you add nodes to a Service Fabric cluster, the application automatically scales, as services are distributed across the new nodes.</span></span>
- <span data-ttu-id="bc15f-153">**服務探索**。</span><span class="sxs-lookup"><span data-stu-id="bc15f-153">**Service discovery**.</span></span> <span data-ttu-id="bc15f-154">Service Fabric 會提供可解析具名服務端點的探索服務。</span><span class="sxs-lookup"><span data-stu-id="bc15f-154">Service Fabric provides a discovery service that can resolve the endpoint for a named service.</span></span>
- <span data-ttu-id="bc15f-155">**無狀態服務和具狀態服務**。</span><span class="sxs-lookup"><span data-stu-id="bc15f-155">**Stateless and stateful services**.</span></span> <span data-ttu-id="bc15f-156">具狀態服務會使用[可靠集合][sf-reliable-collections]，這些集合不僅可取代快取或佇列，還能加以分割。</span><span class="sxs-lookup"><span data-stu-id="bc15f-156">Stateful services use [reliable collections][sf-reliable-collections], which can take the place of a cache or queue, and can be partitioned.</span></span>
- <span data-ttu-id="bc15f-157">**應用程式生命週期管理**。</span><span class="sxs-lookup"><span data-stu-id="bc15f-157">**Application lifecycle management**.</span></span> <span data-ttu-id="bc15f-158">服務可以獨立升級，應用程式不必停機。</span><span class="sxs-lookup"><span data-stu-id="bc15f-158">Services can be upgraded independently and without application downtime.</span></span>
- <span data-ttu-id="bc15f-159">跨機器叢集的**服務協調流程**。</span><span class="sxs-lookup"><span data-stu-id="bc15f-159">**Service orchestration** across a cluster of machines.</span></span>
- <span data-ttu-id="bc15f-160">**更高的密度**以獲得最佳的資源耗用量。</span><span class="sxs-lookup"><span data-stu-id="bc15f-160">**Higher density** for optimizing resource consumption.</span></span> <span data-ttu-id="bc15f-161">單一節點可以裝載多個服務。</span><span class="sxs-lookup"><span data-stu-id="bc15f-161">A single node can host multiple services.</span></span>

<span data-ttu-id="bc15f-162">Service Fabric 可供各種 Microsoft 服務使用，包括 Azure SQL Database、Cosmos DB、Azure 事件中樞等等，因此是經過證實可供建置分散式雲端應用程式的平台。</span><span class="sxs-lookup"><span data-stu-id="bc15f-162">Service Fabric is used by various Microsoft services, including Azure SQL Database, Cosmos DB, Azure Event Hubs, and others, making it a proven platform for building distributed cloud applications.</span></span>
