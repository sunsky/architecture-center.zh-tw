## <a name="why-build-microservices"></a><span data-ttu-id="13061-119">為何要建置微服務？</span><span class="sxs-lookup"><span data-stu-id="13061-119">Why build microservices?</span></span>

<!-- markdownlint-enable MD026 -->

<span data-ttu-id="13061-120">在微服務架構中，應用程式是由數個小型、獨立的服務所組成。</span><span class="sxs-lookup"><span data-stu-id="13061-120">In a microservices architecture, the application is composed of small, independent services.</span></span> <span data-ttu-id="13061-121">以下是微服務的一些定義特性：</span><span class="sxs-lookup"><span data-stu-id="13061-121">Here are some of the defining characteristics of microservices:</span></span>

- <span data-ttu-id="13061-122">每個微服務服務會實作單一商務功能。</span><span class="sxs-lookup"><span data-stu-id="13061-122">Each microservice implements a single business capability.</span></span>
- <span data-ttu-id="13061-123">微服務夠小，只要一小組開發人員即可進行撰寫和維護。</span><span class="sxs-lookup"><span data-stu-id="13061-123">A microservice is small enough that a single small team of developers can write and maintain it.</span></span>
- <span data-ttu-id="13061-124">微服務會在個別的程序中執行，並透過定義完善的 API 或傳訊模式進行通訊。</span><span class="sxs-lookup"><span data-stu-id="13061-124">Microservices run in separate processes, communicating through well-defined APIs or messaging patterns.</span></span>
- <span data-ttu-id="13061-125">微服務不會共用資料存放區或資料結構描述。</span><span class="sxs-lookup"><span data-stu-id="13061-125">Microservices do not share data stores or data schemas.</span></span> <span data-ttu-id="13061-126">每個微服務負責維護自己的資料。</span><span class="sxs-lookup"><span data-stu-id="13061-126">Each microservice is responsible for managing its own data.</span></span>
- <span data-ttu-id="13061-127">微服務具有不同的程式碼基底，而且不會共用原始程式碼。</span><span class="sxs-lookup"><span data-stu-id="13061-127">Microservices have separate code bases, and do not share source code.</span></span> <span data-ttu-id="13061-128">不過會使用共同的公用程式庫。</span><span class="sxs-lookup"><span data-stu-id="13061-128">They may use common utility libraries, however.</span></span>
- <span data-ttu-id="13061-129">每個微服務都可獨立於其他服務進行部署及更新。</span><span class="sxs-lookup"><span data-stu-id="13061-129">Each microservice can be deployed and updated independently of other services.</span></span>

<span data-ttu-id="13061-130">正確完成時，微服務可以提供許多實用優勢：</span><span class="sxs-lookup"><span data-stu-id="13061-130">Done correctly, microservices can provide a number of useful benefits:</span></span>

- <span data-ttu-id="13061-131">**靈活度。**</span><span class="sxs-lookup"><span data-stu-id="13061-131">**Agility.**</span></span> <span data-ttu-id="13061-132">因為微服務為獨自部署，所以可更輕鬆地管理錯誤 (bug) 修正和功能版本。</span><span class="sxs-lookup"><span data-stu-id="13061-132">Because microservices are deployed independently, it's easier to manage bug fixes and feature releases.</span></span> <span data-ttu-id="13061-133">您可以逕自更新服務，而不必重新部署整個應用程式，並於發生錯誤時復原更新。</span><span class="sxs-lookup"><span data-stu-id="13061-133">You can update a service without redeploying the entire application, and roll back an update if something goes wrong.</span></span> <span data-ttu-id="13061-134">在許多傳統應用程式中，如果在應用程式的一個組件中發現錯誤 (bug)，應用程式可能會封鎖整個發行程序；因此，新功能可能延誤，以等候錯誤修正進行整合、測試和發佈。</span><span class="sxs-lookup"><span data-stu-id="13061-134">In many traditional applications, if a bug is found in one part of the application, it can block the entire release process; as a result, new features may be held up waiting for a bug fix to be integrated, tested, and published.</span></span>

- <span data-ttu-id="13061-135">**小型程式碼、小型團隊。**</span><span class="sxs-lookup"><span data-stu-id="13061-135">**Small code, small teams.**</span></span> <span data-ttu-id="13061-136">微服務應小到讓單一功能小組可加以建置、測試及部署。</span><span class="sxs-lookup"><span data-stu-id="13061-136">A microservice should be small enough that a single feature team can build, test, and data-ttu-id="13061-138">在大型單體式應用程式中，程式碼相依性有隨著時間變得紊亂的傾向，以致新增功能時需要碰觸許多位置的程式碼。</span><span class="sxs-lookup"><span data-stu-id="13061-138">In a large monolithic application, there is a tendency over time for code dependencies to become tangled, so that adding a new feature requires touching code in a lot of places.</span></span> <span data-ttu-id="13061-139">微服務架構不會共用程式碼或資料存放區，因而降低相依性，並可讓您更輕鬆地新增功能。</span><span class="sxs-lookup"><span data-stu-id="13061-139">By not sharing code or data stores, a microservices architecture minimizes class="sxs-lookup"><span data-stu-id="13061-143">But the point is that large groups tend be less productive, because communication is slower, management overhead goes up, and agility - <span data-ttu-id="13061-144">**混用技術**。</span><span class="sxs-lookup"><span data-stu-id="13061-144">**Mix of technologies**.</span></span> <span data-ttu-id="13061-145">小組可以視情況使用混合的技術堆疊，以挑選出最適合其服務使用的技術。</span><span class="sxs-lookup"><span data-stu-id="13061-145">Teams can pick the technology that best fits their service, using a mix of technology stacks as appropriate.</span></span>

- <span data-ttu-id="13061-146">**復原功能**。</span><span class="sxs-lookup"><span data-stu-id="13061-146">**Resiliency**.</span></span> <span data-ttu-id="13061-147">如果有個別微服務無法使用，只要有任何上游微服務是設計用來正確地處理錯誤 (例如，藉由實作線路中斷)，就不會中斷整個應用程式。</span><span class="sxs-lookup"><span data-stu-id="13061-147">If an individual microservice becomes unavailable, it won't disrupt the entire application, as long as any upstream microservices are designed to handle faults correctly (for example, by implementing circuit breaking).</span></span>

- <span data-ttu-id="13061-148">**延展性**。</span><span class="sxs-lookup"><span data-stu-id="13061-148">**Scalability**.</span></span> <span data-ttu-id="13061-149">微服務架構可讓每個微服務獨立於其他服務進行調整。</span><span class="sxs-lookup"><span data-stu-id="13061-149">A microservices architecture allows each microservice to be scaled independently of the others.</span></span> <span data-ttu-id="13061-150">這可讓您相應放大需要更多資源的子系統，而不需相應放大整個應用程式。</span><span class="sxs-lookup"><span data-stu-id="13061-150">That lets you scale out subsystems that require more resources, without scaling out the entire application.</span></span> <span data-ttu-id="13061-151">如果您在容器內部署服務，您也可以將大量微服務封裝到單一主機，以達到更有效的資源使用率。</span><span class="sxs-lookup"><span data-stu-id="13061-151">If you deploy services inside containers, you can also pack a higher density of microservices onto a single host, which allows for more efficient utilization of resources.</span></span>

- <span data-ttu-id="13061-152">**資料隔離**。</span><span class="sxs-lookup"><span data-stu-id="13061-152">**Data isolation**.</span></span> <span data-ttu-id="13061-153">您可更輕鬆地執行結構描述更新，因為只有單一微服務受到影響。</span><span class="sxs-lookup"><span data-stu-id="13061-153">It is much easier to perform schema updates, because only a single microservice is affected.</span></span> <span data-ttu-id="13061-154">在單體式應用程式中，結構描述更新可能變得非常具挑戰性，因為應用程式的不同組件可能會觸碰相同的資料，以致任何結構描述變化都有風險。</span><span class="sxs-lookup"><span data-stu-id="13061-154">In a monolithic application, schema updates can become very challenging, because different parts of the application may all touch the same data, making any alterations to the schema risky.</span></span>

## <a name="no-free-lunch"></a><span data-ttu-id="13061-155">天下沒有白吃的午餐</span><span class="sxs-lookup"><span data-stu-id="13061-155">No free lunch</span></span>
