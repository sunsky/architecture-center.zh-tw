# <a name="autoscaling"></a><span data-ttu-id="dc40c-103">自動調整</span><span class="sxs-lookup"><span data-stu-id="dc40c-103">Autoscaling</span></span>
[!INCLUDE [header](../_includes/header.md)]

<span data-ttu-id="dc40c-104">自動調整是以動態方式配置資源以符合效能需求的程序。</span><span class="sxs-lookup"><span data-stu-id="dc40c-104">Autoscaling is the process of dynamically allocating resources to match performance requirements.</span></span> <span data-ttu-id="dc40c-105">隨著工作量的成長，應用程式可能需要額外的資源來保有所需的效能層級及滿足服務等級協定 (SLA)。</span><span class="sxs-lookup"><span data-stu-id="dc40c-105">As the volume of work grows, an application may need additional resources to maintain the desired performance levels and satisfy service-level agreements (SLAs).</span></span> <span data-ttu-id="dc40c-106">當需要降低而不再需要其他資源，可以將它們解除配置以將成本降至最低。</span><span class="sxs-lookup"><span data-stu-id="dc40c-106">As demand slackens and the additional resources are no longer needed, they can be de-allocated to minimize costs.</span></span>

<span data-ttu-id="dc40c-107">自動調整會充分利用雲端裝載環境的彈性，以降低管理負擔。</span><span class="sxs-lookup"><span data-stu-id="dc40c-107">Autoscaling takes advantage of the elasticity of cloud-hosted environments while easing management overhead.</span></span> <span data-ttu-id="dc40c-108">它可減少操作員持續監視系統效能的需求，並做出有關新增或移除資源的決策。</span><span class="sxs-lookup"><span data-stu-id="dc40c-108">It reduces the need for an operator to continually monitor the performance of a system and make decisions about adding or removing resources.</span></span>

<span data-ttu-id="dc40c-109">應用程式有兩種主要方式可調整：</span><span class="sxs-lookup"><span data-stu-id="dc40c-109">There are two main ways that an application can scale:</span></span> 

* <span data-ttu-id="dc40c-110">**垂直縮放比例**，也稱為相應增加和相應減少，表示變更資源的容量。</span><span class="sxs-lookup"><span data-stu-id="dc40c-110">**Vertical scaling**, also called scaling up and down, means changing the capacity of a resource.</span></span> <span data-ttu-id="dc40c-111">例如，您可以將應用程式移動為較大的 VM 大小。</span><span class="sxs-lookup"><span data-stu-id="dc40c-111">For example, you could move an application to a larger VM size.</span></span> <span data-ttu-id="dc40c-112">垂直調整通常要求在重新部署時系統必須暫時無法使用。</span><span class="sxs-lookup"><span data-stu-id="dc40c-112">Vertical scaling often requires making the system temporarily unavailable while it is being redeployed.</span></span> <span data-ttu-id="dc40c-113">因此，將垂直調整自動化較不常見。</span><span class="sxs-lookup"><span data-stu-id="dc40c-113">Therefore, it's less common to automate vertical scaling.</span></span>
* <span data-ttu-id="dc40c-114">**水平調整**，也稱為向外和向內調整，表示新增或移除資源的執行個體。</span><span class="sxs-lookup"><span data-stu-id="dc40c-114">**Horizontal scaling**, also called scaling out and in, means adding or removing instances of a resource.</span></span> <span data-ttu-id="dc40c-115">在佈建這些新資源時，應用程式會繼續執行而不中斷。</span><span class="sxs-lookup"><span data-stu-id="dc40c-115">The application continues running without interruption as new resources are provisioned.</span></span> <span data-ttu-id="dc40c-116">佈建程序完成時，解決方案即會部署在這些額外的資源上。</span><span class="sxs-lookup"><span data-stu-id="dc40c-116">When the provisioning process is complete, the solution is deployed on these additional resources.</span></span> <span data-ttu-id="dc40c-117">如果要求降低，則可以完全關閉及取消配置這些額外的資源。</span><span class="sxs-lookup"><span data-stu-id="dc40c-117">If demand drops, the additional resources can be shut down cleanly and deallocated.</span></span> 

<span data-ttu-id="dc40c-118">許多雲端系統 (包括 Microsoft Azure) 都支援自動的垂直調整。</span><span class="sxs-lookup"><span data-stu-id="dc40c-118">Many cloud-based systems, including Microsoft Azure, support automatic horizontal scaling.</span></span> <span data-ttu-id="dc40c-119">本文的其餘部分著重於水平調整。</span><span class="sxs-lookup"><span data-stu-id="dc40c-119">The rest of this article focuses on horizontal scaling.</span></span>

> [!NOTE]
> <span data-ttu-id="dc40c-120">自動調整大多適用於計算資源。</span><span class="sxs-lookup"><span data-stu-id="dc40c-120">Autoscaling mostly applies to compute resources.</span></span> <span data-ttu-id="dc40c-121">雖然可以水平調整資料庫或訊息佇列，這通常牽涉到一般不會自動化的[資料分割][data-partitioning]。</span><span class="sxs-lookup"><span data-stu-id="dc40c-121">While it's possible to horizontally scale a database or message queue, this usually involves [data partitioning][data-partitioning], which is generally not automated.</span></span>

<span data-ttu-id="dc40c-123">自動調整策略通常牽涉到下列部分：</span><span class="sxs-lookup"><span data-stu-id="dc40c-123">An autoscaling strategy typically involves the following pieces:</span></span>

* <span data-ttu-id="dc40c-234">請記住，自動調整可能不是處理工作負載中突發高載狀況的最適當機制。</span><span class="sxs-lookup"><span data-stu-id="dc40c-234">Keep in mind that autoscaling might not be the most appropriate mechanism to handle a sudden burst in workload.</span></span> <span data-ttu-id="dc40c-235">佈建並啟動新服務執行個體或將資源新增到系統都需要花時間，而當這些額外資源可供使用時，尖峰需求可能已過。</span><span class="sxs-lookup"><span data-stu-id="dc40c-235">It takes time to provision and start new instances of a service or add resources to a system, and the peak demand may have passed by the time these additional resources have been made available.</span></span> <span data-ttu-id="dc40c-236">在這種情況下，節流服務可能比較適合。</span><span class="sxs-lookup"><span data-stu-id="dc40c-236">In this scenario, it may be better to throttle the service.</span></span> <span data-ttu-id="dc40c-237">如需詳細資訊，請參閱[節流模式][throttling]。</span><span class="sxs-lookup"><span data-stu-id="dc40c-237">For more information, see the [Throttling Pattern][throttling].</span></span>
* <span data-ttu-id="dc40c-238">相反地，如果您需要在交易量快速波動時足以處理所有要求的能力，而且成本不是主要考量因素，那麼請考慮使用積極的自動調整策略，以更快速啟動額外的執行個體。</span><span class="sxs-lookup"><span data-stu-id="dc40c-238">Conversely, if you do need the capacity to process all requests when the volume fluctuates rapidly, and cost isn't a major contributing factor, consider using an aggressive autoscaling strategy that starts additional instances more quickly.</span></span> <span data-ttu-id="dc40c-239">您也可以使用排程原則，在最大負載來臨前預先啟動足量的執行個體。</span><span class="sxs-lookup"><span data-stu-id="dc40c-239">You can also use a scheduled policy that starts a sufficient number of instances to meet the maximum load before that load is expected.</span></span>
* <span data-ttu-id="dc40c-240">自動調整機制應該監視自動調整程序，並記錄每個自動調整事件的詳細資料 (觸發的事件、加入或移除哪些資源，以及發生時間)。</span><span class="sxs-lookup"><span data-stu-id="dc40c-240">The autoscaling mechanism should monitor the autoscaling process, and log the details of each autoscaling event (what triggered it, what resources were added or removed, and when).</span></span> <span data-ttu-id="dc40c-241">如果您建立自訂自動調整機制，請確定它包含這項功能。</span><span class="sxs-lookup"><span data-stu-id="dc40c-241">If you create a custom autoscaling mechanism, ensure that it incorporates this capability.</span></span> <span data-ttu-id="dc40c-242">分析資訊，以協助測量自動調整策略的有效性，並視需要加以微調。</span><span class="sxs-lookup"><span data-stu-id="dc40c-242">Analyze the information to help measure the effectiveness of the autoscaling strategy, and tune it if necessary.</span></span> <span data-ttu-id="dc40c-243">您可以在使用模式變得更明顯時短時間內進行調整，以及在業務量拓展或對應用程式的需求演變時長期進行調整。</span><span class="sxs-lookup"><span data-stu-id="dc40c-243">You can tune both in the short term, as the usage patterns become more obvious, and over the long term, as the business expands or the requirements of the application evolve.</span></span> <span data-ttu-id="dc40c-244">如果應用程式達到定義的自動調整上限，機制也可以警告操作員，讓操作員視需要手動啟動其他資源。</span><span class="sxs-lookup"><span data-stu-id="dc40c-244">If an application reaches the upper limit defined for autoscaling, the mechanism might also alert an operator who could manually start additional resources if necessary.</span></span> <span data-ttu-id="dc40c-245">請注意，在這種情況下，操作員可能也需在工作負載減輕後，手動移除這些資源。</span><span class="sxs-lookup"><span data-stu-id="dc40c-245">Note that, under these circumstances, the operator may also be responsible for manually removing these resources after the workload eases.</span></span>
