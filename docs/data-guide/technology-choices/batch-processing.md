# <a name="choosing-a-batch-processing-technology-in-azure"></a><span data-ttu-id="a107c-102">在 Azure 中選擇批次處理技術</span><span class="sxs-lookup"><span data-stu-id="a107c-102">Choosing a batch processing technology in Azure</span></span>

<span data-ttu-id="a107c-103">巨量資料解決方案通常使用長時間執行的批次作業來篩選、彙總和準備資料以供分析。</span><span class="sxs-lookup"><span data-stu-id="a107c-103">Big data solutions often use long-running batch jobs to filter, aggregate, and otherwise prepare the data for analysis.</span></span> <span data-ttu-id="a107c-104">這些作業通常包括從可擴充儲存體 (例如 HDFS、Azure Data Lake Store 與 Azure 儲存體) 讀取原始檔並加以處理，以及將輸出寫入到可擴充儲存體中的新檔案。</span><span class="sxs-lookup"><span data-stu-id="a107c-104">Usually these jobs involve reading source files from scalable storage (like HDFS, Azure Data Lake Store, and Azure Storage), processing them, and writing the output to new files in scalable storage.</span></span>
